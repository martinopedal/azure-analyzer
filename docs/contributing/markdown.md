# Markdown contributor guide

This repo runs three checks on every Markdown change. All three live in a single
workflow: `.github/workflows/markdown-check.yml`.

| Job        | Tool                  | Config                       | Scope                                |
| ---------- | --------------------- | ---------------------------- | ------------------------------------ |
| `lint`     | markdownlint-cli2     | `.markdownlint-cli2.jsonc`   | All `**/*.md`, minus the ignore list |
| `links`    | lychee                | `.lychee.toml`               | PR: changed `*.md`; schedule/manual: full `**/*.md` (7d cache) |
| `em-dash`  | ripgrep               | (workflow inline)            | Changed Markdown on PRs, minus ephemeral agent-state trees |

## Run locally

```powershell
# Lint (Node 20+; uses npx so no global install needed)
npx -y markdownlint-cli2 "**/*.md"

# Links (requires lychee binary; install via cargo install lychee)
lychee --config .lychee.toml --cache --max-cache-age 7d "./**/*.md"

# PR-like scope (changed Markdown only)
$base = git merge-base origin/main HEAD
$targets = git diff --name-only --diff-filter=ACMR $base HEAD -- '**/*.md'
if ($targets) { lychee --config .lychee.toml --cache --max-cache-age 7d $targets }

# Em-dash policy (requires ripgrep)
rg -l -e ([char]0x2014) -e ([char]0x2013) --glob '*.md'
```

## Em-dash policy

This repo forbids em-dash (`U+2014`) and en-dash (`U+2013`) in Markdown. Use one
of the alternatives:

| Forbidden | Use instead         |
| --------- | ------------------- |
| ` foo - bar `  | ` foo - bar ` (space-hyphen-space) |
| ` 1 - 5 `      | ` 1 to 5 ` or `1..5`               |

The PR gate only fails on em-dashes introduced by your PR. The scheduled run
emits a backlog list to the workflow summary so historical hits can be cleaned
up incrementally.

## Adding an exception

### Lint exception (markdownlint-cli2)

For a one-line exception, use an inline disable directive at the top of the
offending block. For a whole file or directory, add it to the `ignores` list in
`.markdownlint-cli2.jsonc`. Justify the exception with a comment.

### Link exception (lychee)

For a single URL pattern, append it to `exclude` in `.lychee.toml`. For an
entire path, append to `exclude_path`. Always leave a comment explaining why.
Auto-generated docs and ephemeral agent state (squad inbox, copilot audits) are
already excluded.

### Em-dash exception

There are no exceptions within the policy scan scope. Replace the character. If
you have a quoting need that genuinely requires the typographic dash (rare),
open an issue first.

## How the three checks interact

- The three jobs run in parallel; a failure in one does not block the others
  from reporting.
- `lint` and `em-dash` are deterministic and should never flake.
- `links` uses a 7-day cache, 5 in-process retries with a 10s wait, and a
  workflow-level `nick-fields/retry` wrapper for catastrophic transient
  failures (TLS handshake, DNS). Retry attempts clear `.lycheecache` so a
  cached transient 5xx does not poison the next attempt. PR runs scope lychee
  to changed Markdown files only (less rate-limit pressure), while
  schedule/manual runs scan the full Markdown corpus to keep link-rot visible.
  The workflow also passes `GITHUB_TOKEN` to lychee for more reliable GitHub
  URL checks.
- Auto-generated docs (`docs/reference/permissions/`, `docs/reference/tool-catalog*.md`,
  and the `docs/consumer/` mirrors) are excluded from `lint` because they are
  regenerated by tooling. Validate them upstream in the generator.
- Ephemeral agent state (`.copilot/audits/`, `.copilot/status/`,
  `.copilot/session-state/`, `.squad/decisions/inbox/`, `.atlas-stash/`) is
  excluded from `lint`, `links`, and PR `em-dash` gating because lifetime is
  hours and stale references are expected.
