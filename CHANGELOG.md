- `report-template.html`, `New-HtmlReport.ps1`: Add client-side CSV export button — exports currently visible (filtered) findings as a CSV download from embedded data; no re-run needed to get findings into a spreadsheet.

### Fixed
- **`pr-advisory-gate` degraded no-op is non-blocking again (#182)**: `.github/workflows/pr-advisory-gate.yml` now emits `gate-state=success` (not `failure`) when the advisory step degrades in its catch block, and writes `skip-reason=degraded: <sanitized error>` to `GITHUB_OUTPUT`. Also hardened `modules/shared/Get-CopilotReviewFindings.ps1` to normalize owner/repo inputs defensively so GraphQL `name` is always repo-name only (never `owner/repo`), preventing doubled-slug failures like `martinopedal/martinopedal/azure-analyzer`. Added regression coverage in `tests/shared/Get-CopilotReviewFindings.Tests.ps1` and workflow guardrails in `tests/workflows/PRAdvisoryGate.Tests.ps1`.
- **`sentinel-coverage` post-merge rubber-duck follow-up to PR #180 (commit `cef1d30`)**: 3-of-3 frontier-model consensus (`claude-opus-4.7` + `gpt-5.3-codex` + `goldeneye`, verdict APPROVE-WITH-CONDITIONS) flagged five gaps, all addressed in `modules/Invoke-SentinelCoverage.ps1`. (1) **Sentinel onboarding probe** — added a Step 0 `GET .../onboardingStates/default` call; 404 → `Skipped` (workspace not onboarded), 401/403 → `Skipped` (insufficient role; user may have LA Reader but not Sentinel Reader), other failures fall through to best-effort detection so we don't false-Skip on a transient ARM hiccup. Replaces the old `alertRules`-404 skip premise (alertRules returns 200 + empty array on a non-onboarded workspace, so the skip almost never fired). (2) **`nextLink` pagination** — new `Invoke-SentinelGetPaged` helper walks `value.nextLink` for all five list calls (`alertRules`, `dataConnectors`, `watchlists`, `watchlistItems`, `savedSearches`); first-page non-200 returns the raw response with empty `Items`, subsequent-page failures degrade to best-effort with a sanitized warning. Prevents under-counting on workspaces with >100 rules / connectors / watchlist items. (3) **Connector enabled-state filter** — new `Test-ConnectorEnabled` helper inspects `properties.dataTypes.*.state` (handling both flat-object and array shapes) and the few-connectors detection now counts only those with at least one `Enabled` data type, not all registered connectors. Eliminates the false-negative where a workspace registers 5 connectors but has 0 actually streaming telemetry. (4) **`LookbackDays` wired in** — the parameter was declared but unused; it now drives the disabled-rule staleness threshold (replaces the hard-coded `$script:DisabledRuleStaleDays = 7`) so callers can dial the sensitivity. (5) **URL-encoded watchlist alias** — `[uri]::EscapeDataString($alias)` is applied before the alias is interpolated into the `watchlists/{alias}/watchlistItems` URI, preventing 404s on aliases containing spaces or other reserved characters. (6) **Latent ISO-8601 date-parse bug fixed** — `ConvertFrom-Json` in PS 7+ auto-deserializes ISO-8601 strings to `DateTime` (System.Text.Json behavior); the `[string]` cast then rendered them in the current culture, which `[datetime]::Parse` rejected → `$age` became `$null` → fell into the "no lastModifiedUtc" arm and over-flagged. The parser now branches on `$rawValue -is [datetime]` first and only calls `[datetime]::Parse` on actual string inputs with explicit `InvariantCulture` + `RoundtripKind` styles. New Pester coverage in `tests/wrappers/Invoke-SentinelCoverage.Tests.ps1` (6 new contexts, ~18 new tests) covers the onboarding-probe 404/403 paths, pagination across two pages, the enabled-state connector filter (registered≠enabled), `LookbackDays` threshold semantics, and the URL-encoded alias on the wire (asserted via `$Uri.AbsoluteUri` since `[Uri].ToString()` decodes `%20` → space). Pester baseline: **1019/1019 green** (was 1001/1001 pre-follow-up).
