# Changelog

All notable changes to azure-analyzer will be documented here.

## [Unreleased]

### Added
- **Universal advisory review gate on every squad PR (#109)**: New `.github/workflows/pr-advisory-gate.yml` workflow runs on `pull_request_target` (opened, ready_for_review, synchronize) with a fork-skip guard, minimal `pull-requests: write` + `contents: read` permissions, and a PR-keyed concurrency group with `cancel-in-progress: true`. Only runs when the PR author is a squad agent or bot (login ends in `[bot]`, matches a built-in squad identity, or is listed in the `SQUAD_AGENT_LOGINS` repo variable), human PRs are skipped. Calls `modules/shared/Invoke-PRAdvisoryGate.ps1`, which builds the 3-model triage bundle path, then posts or updates a single advisory comment on the PR under the `<!-- squad-advisory -->` marker so re-runs update in place. Findings are tagged per the #108 severity taxonomy; untagged findings are auto-tagged `[correctness]` (fail-safe). Verdict escalates to `blockers` automatically when any veto-class finding lands. Non-blocking by design, the human / Copilot reviewer always has final say. Disable repo-wide via `SQUAD_ADVISORY_GATE=0`. Covered by 23 Pester tests in `tests/workflows/PRAdvisoryGate.Tests.ps1` (squad-author detection, severity auto-tagging, comment formatting, disable + human-author short-circuits, workflow YAML safety: SHA-pinned action, fork-skip guard, no `${{ }}` interpolation into bash, concurrency, repo-variable default). Closes #109.
- **Auto-resolve PR review threads on revision (#106)**: New `pr-auto-resolve-threads.yml` workflow runs on `pull_request_target` (synchronize, ready_for_review, reopened) and `pull_request_review` (submitted, edited), with a fork-skip guard. It calls `modules/shared/Resolve-PRReviewThreads.ps1`, which lists unresolved review threads via the GitHub GraphQL API (paginated), fetches commits added AFTER each thread was created, and parses each commit's diff hunks. When a commit's RIGHT-side hunk overlaps the thread's file + line range, the thread is resolved via the `resolveReviewThread` mutation and a short reply is posted on the thread linking the addressing commit SHA. Threads the new commits did NOT touch stay open, the reviewer decides. Reply uses the `<!-- squad-auto-resolve-thread -->` marker for in-place updates per the #113 notification-hygiene convention; existing markers are detected so overlapping runs do not duplicate replies. Workflow uses minimal `pull-requests: write` permission, gated by `SQUAD_AUTO_RESOLVE_THREADS` repo variable (set to `0` to disable). Covered by 7 Pester tests in `tests/shared/Resolve-PRReviewThreads.Tests.ps1`. The `.copilot/copilot-instructions.md` cloud agent PR review contract section now documents the auto-resolve behavior.
- **Review severity taxonomy for PR review feedback (#108)**: Reviewers (Copilot, the 3-model gate, and humans) must now prefix each PR review finding with one of four severity tags: `[blocker]`, `[correctness]`, `[style]`, `[nit]`. Only `[blocker]` and `[correctness]` block merge and trigger Reviewer Rejection Lockout; `[style]` and `[nit]` are advisory and merge proceeds. Gate-pass criteria documented as ordered rules: (1) no `[blocker]` or `[correctness]` finding from any reviewer (absolute veto), AND (2) either 2-of-3 APPROVE OR all `REQUEST_CHANGES` findings are `[style]`/`[nit]` only. Untagged findings are treated as `[correctness]` (fail-safe). Taxonomy contract added to `.copilot/copilot-instructions.md` (full reviewer instructions, examples per tag, ambiguity rule). PR-review-handling section in `.squad/templates/issue-lifecycle.md` updated to mirror the taxonomy and the spawn-prompt convention for the 3-model gate. **Doc/policy-only change**: `pr-review-gate.yml` workflow code is **not** modified here; auto-classification of untagged findings is tracked under #109. Stops the gate from burning premium tokens on quote-style nits and keeps the Lockout signal sharp on real defects.
- **Pre-PR self-review template for squad agents (#110)**: Every squad agent must now populate a `## Self-review` section in the PR body before calling `gh pr create`. This is a **policy gate (CI enforcement deferred to follow-up)**, enforced manually by the Squad coordinator and PR reviewers until the CI check ships. The block requires a 3-bullet Diff summary, Risks considered (including an explicit "out of scope on purpose" line), and Testing evidence (actual command + pass/fail counts). Enforcement contract added to `.copilot/copilot-instructions.md`. PR description template in `.squad/templates/issue-lifecycle.md` updated to embed the block. Brief pointers added to `.github/agents/squad.agent.md` and `.squad/templates/squad.agent.md` so the Issue → PR → Merge lifecycle references the new gate. Cuts reviewer orientation time, makes the advisory review gate (#109) cheaper, and creates an audit trail for Reviewer Rejection Lockout disputes. Optional CI workflow check (rejecting PRs missing the header) is intentionally deferred, not part of MVP.
- **SBOM generation + pinned versions for all Installer-downloaded tools (#102)**: Version pins with SHA-256 checksums now live in `tools/install-manifest.json`. CLI tools downloaded via direct URLs (Linux azqr, gitleaks, trivy, scorecard) have their SHA-256 verified after download via new `Get-FileHash256` and `Test-InstallManifestHash` functions in `Installer.ps1`. Package-manager installs (winget/brew/pipx) delegate checksum verification to the package manager itself (documented in `pinningNote` per tool). Each release now includes a **CycloneDX 1.5 SBOM** (`sbom.json`) generated by `tools/Generate-SBOM.ps1`, listing all bundled tools with exact versions, SHA-256 checksums, upstream URLs, and dependency relationships. New CI workflow (`.github/workflows/ci.yml`) runs Pester tests + SBOM generation on every PR. New release workflow (`.github/workflows/release.yml`) auto-generates and attaches the SBOM to each tag. Pester tests (`tests/shared/Installer.Tests.ps1`) cover hash computation, mismatch detection, and placeholder handling. Closes #102.
- Added CI failure watchdog (workflow + local helper) - #104
- **FindingRow schema validation at normalizer boundary (#99)**: All normalizers now validate findings at construction time via `New-FindingRow` factory. Validation enforces required fields (Id, Source, EntityId, EntityType, Title, Compliant, Provenance.RunId), checks Severity/Platform/EntityType against allowed values, and verifies EntityId canonicalization. Invalid rows are dropped with a sanitized warning logged to stderr; validation failures are tracked via `Get-SchemaValidationFailures` for observability. New `Test-FindingRow` function supports both silent (returns bool + error array) and strict modes (throws exception). Covered by 21 comprehensive tests in `tests/shared/Schema-Validation.Tests.ps1`.
- **Pre-commit hook infrastructure for gitleaks + zizmor (#103)**: Opt-in `hooks/pre-commit.ps1` runs gitleaks on staged changes and zizmor on modified workflow files. `tools/Install-PreCommitHook.ps1` installer creates a cross-platform hook (Windows .cmd shim / Unix symlink). Hook gracefully skips with warnings if tools aren't installed. Blocks commits when issues are found (bypassable with `--no-verify`). Covered by 16 Pester tests in `tests/hooks/PreCommit.Tests.ps1`. Added Developer Setup section to README.md with install instructions.

- **Wrapper error-path reliability coverage across all 17 wrappers (#98 / PR #120)**: Added broad error-path tests and wrapper hardening so tool failures degrade gracefully with deterministic `Status/Message` behavior and sanitized diagnostics.

### Fixed
- **PR advisory gate hardening (#139):** `Invoke-PRAdvisoryGate.ps1` now preserves `[security]` tags and treats `[security]` as veto-class for blocker escalation, parses paginated PR comments with `gh api --paginate --slurp` safely across multi-page responses, and resolves duplicate advisory markers by updating the newest marker comment ID. `.github/workflows/pr-advisory-gate.yml` now enforces `timeout-minutes: 5` and explicitly skips draft PRs until ready-for-review. Added focused Pester coverage in `tests/workflows/PRAdvisoryGate.Tests.ps1` for security-tag handling, slurped pagination parsing, duplicate-marker newest-update behavior, and workflow timeout enforcement.
- **Security:** All error messages written to disk (JSON, HTML, logs) are now sanitized via `Remove-Credentials` to prevent credentials, tokens, connection strings, or SAS URIs from being written to disk. Applies to all tool wrappers and the main orchestrator (#100)
- **Security: PR auto-resolve workflow now runs trusted base-ref code (PR #137 gate fix)**: `.github/workflows/pr-auto-resolve-threads.yml` previously used `pull_request` and ran the PR head's copy of `Resolve-PRReviewThreads.ps1` while holding `pull-requests: write`, allowing a fork PR to substitute a malicious resolver and exfiltrate the workflow token. Switched the trigger to `pull_request_target` so the workflow runs in the base ref's context with the base token (and `actions/checkout` defaults to the base SHA when no `ref` is supplied). Added a defense-in-depth guard that skips the job entirely when `github.event.pull_request.head.repo.full_name != github.repository`. Hardened the resolver in `modules/shared/Resolve-PRReviewThreads.ps1`: deletion-only diff hunks (`@@ -a,b +c,0 @@`) no longer fabricate a right-side line and falsely match unrelated threads; files arriving without a `patch` (binary, rename, oversized) are now skipped with a warning instead of being marked as fully touched; `reviewThreads` GraphQL query now paginates via `pageInfo { hasNextPage endCursor }` so PRs with >100 threads are fully covered; reply POSTs scan existing thread comments for the `<!-- squad-auto-resolve-thread -->` marker and skip duplicates so overlapping concurrent runs no longer post twice. Pester coverage extended to 7 tests in `tests/shared/Resolve-PRReviewThreads.Tests.ps1` (deletion-only hunk skip, missing patch skip, pagination across multiple pages, marker-already-present skip).
- **Security:** `Invoke-Trivy.ps1` now sanitizes non-success warning/message paths for missing-report and JSON-parse failures; `Invoke-Falco.ps1` now self-loads `Sanitize.ps1` with local fallback (parallel-safe) and sanitizes emitted alert/log detail fields plus log-collection warnings.
- **Security:** Added parallel/runspace-safe `Sanitize.ps1` self-load + local `Remove-Credentials` fallback in `Invoke-AzureCost.ps1`, `Invoke-DefenderForCloud.ps1`, `Invoke-Kubescape.ps1`, and the orchestrator `$runnerBlock` in `Invoke-AzureAnalyzer.ps1` to prevent `CommandNotFoundException` in `ForEach-Object -Parallel` error paths. Also normalized minor Trivy formatting (`} finally {`, indentation) with no behavior change.
