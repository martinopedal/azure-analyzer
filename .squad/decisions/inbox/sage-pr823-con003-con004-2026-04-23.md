## Decision

Take over the cloud-agent draft PR #823 and drive it to green + merged. Address all 10 unresolved Copilot review threads, tighten the CON-003 / CON-004 ratchet regexes so they actually catch inline throws and bare `ShouldProcess` mentions, and add two CHANGELOG bullets.

## What landed

1. **Shim alignment (17 wrappers).** The per-wrapper `New-FindingError` shim now mirrors the canonical `modules/shared/Errors.ps1` contract: Category enum validation (via `Write-Error -ErrorAction Stop` to avoid re-introducing a raw throw), `Remove-Credentials` on Reason / Remediation / Details, and `TimestampUtc`. Previously the shim silently dropped validation and sanitization when the shared module was not yet dot-sourced.
2. **Ratchet tightening.**
   - `Get-RawThrowCount` regex: `(?<![a-zA-Z0-9_\-])throw\s+["']` - catches any raw `throw '...'` including inline guard clauses. Old regex matched only line-start throws and missed `if (-not $ctx) { throw 'No Az context' }`.
   - CON-004 assertion: requires BOTH `SupportsShouldProcess(\s*=\s*\$true)?(\s*[,\)])` AND `\$PSCmdlet\.ShouldProcess\s*\(`. Old regex passed on `SupportsShouldProcess=$false` or a bare comment mention of `ShouldProcess`.
3. **Residual raw throws migrated (7).** 6x `if (-not $ctx) { throw 'No Az context' }` guards + 1 inline throw in `Invoke-ADORepoSecrets` converted to `Write-Error -ErrorAction Stop`. Equivalent terminating-error semantics inside the surrounding try/catch, but satisfies the tightened regex.
4. **Powerpipe signal preservation.** `Invoke-Powerpipe` now emits the sanitized CLI output via `Write-Verbose` immediately before throwing so the Details payload survives `Format-FindingErrorMessage` (which does not itself emit a Details field).
5. **CI hygiene.** The two step-level `continue-on-error: true` entries added to `ci.yml` by PR #861 now carry the `# tracked: martinopedal/azure-analyzer#604` marker required by `WorkflowHygiene.Tests.ps1`. Without this the test goes red on all three OSes.
6. **CHANGELOG.** Collapsed dual `## Unreleased` blocks created by the mid-PR release-please cut of `[1.1.1]`, replaced em-dashes with ASCII in new content, and split the PR #823 entry into CON-003 (#626) and CON-004 (#627) bullets.
7. **Review threads.** All 10 unresolved Copilot threads resolved via `gh api graphql resolveReviewThread` after the fixes shipped, plus a consolidated summary comment on the PR citing the commit SHA.

## Policy reinforcement

- Anyone adding a future `continue-on-error: true` in any workflow under `.github/workflows/` MUST add `# tracked: martinopedal/azure-analyzer#604` on the immediately-preceding non-blank line, or `WorkflowHygiene.Tests.ps1` will fail on all three OSes.
- Any new wrapper MUST NOT use raw `throw '...'`. Use `New-FindingError` + `Format-FindingErrorMessage` + `throw`, or `Write-Error -ErrorAction Stop` inside a guard clause. Enforced by `Get-RawThrowCount` with the new `(?<![a-zA-Z0-9_\-])throw\s+["']` regex and `RawThrowBaseline = @{}`.
- Any side-effecting wrapper that declares `[CmdletBinding(SupportsShouldProcess=$true)]` MUST gate all mutations with `$PSCmdlet.ShouldProcess(...)`. Enforced by CON-004 with the tightened regex pair.
