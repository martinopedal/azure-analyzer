### 20260418172749 UTC: User directive - issue-first workflow
**By:** martinopedal (via Copilot)
**What:** We do not ship PRs we ad-hoc - we work on issues. When an issue is fully planned out (acceptance criteria, design notes, scope confirmed), then we create code and get to work based on the plan from the issue. PRs reference and implement an issue plan, never the other way around.
**Why:** Consistency across the squad. The issue is the contract; the PR is the implementation of that contract. Forces planning before code.
