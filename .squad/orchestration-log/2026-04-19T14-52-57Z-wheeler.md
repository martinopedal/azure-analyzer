## Root Cause

The `ci-failure-watchdog.yml` workflow was missing the `workflows:` key on the `workflow_run` trigger. This caused the workflow to not properly register event listeners for failed workflow runs, leading to intermittent failures and incorrect event routing.

## Fix Applied

PR #154 added the `workflows:` key to the `workflow_run` trigger block:

```yaml
on:
  workflow_run:
    workflows: ['Analyze']
    types: ['completed']
```

  - Run 2: `conclusion=success`, `event=workflow_run` ✓
- Previous rate: 50% failure (intermittent event payload parsing)
