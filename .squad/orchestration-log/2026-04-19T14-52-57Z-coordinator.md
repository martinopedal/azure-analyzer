## Root Cause Identified Later (Wheeler)

Deeper analysis by Wheeler revealed the true root cause was missing `workflows:` key on the `workflow_run` trigger, not step-level gating logic. This prevented proper event listener registration.

## Outcome

PR #153 was a valid first attempt at stabilization but did not address the underlying event registration issue. Wheeler's PR #154 superseded it with the correct fix. Both are in the git history; the later commit (0f287ad) represents the complete solution.

## Learnings

1. Event trigger configuration takes precedence over runtime gate logic.
2. GitHub Actions `workflow_run` payloads require explicit `workflows:` key to listen for specific triggering workflows.
3. Step-level gates can mask root-cause issues; always verify event registration first.
