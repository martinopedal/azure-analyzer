## Technical Pivot 1: Orphaned CopilotTriage Wrapper (PR #1082)

**Problem:**  
Execution tests in `tests/Wrappers/Invoke-CopilotTriage.Execution.Tests.ps1` expected Status='Failed' when AI triage failed. But the actual code path is orphaned (module disabled in manifest; enabled=false). Ubuntu CI couldn't satisfy github-copilot-sdk prereqs, so the wrapper short-circuited to Status='Skipped' — tests failed.

**False solutions rejected:**
- ~~Add hidden `-SkipPrerequisiteChecks` bypass param~~ (creates production-callable bypass on AI-triage code; security risk)
- ~~Lower test expectations to Skipped~~ (masks real failure scenarios; test smell)

**Resolution:**  
**Defensive-depth contract enforcement.** Replaced 4 execution tests with 6 architectural + structural guardrails:
- Assert `modules/Invoke-CopilotTriage.ps1` is disabled by design in manifest
- Validate wrapper never reaches production code path
- Document contract: "Orphaned by design; re-enable only with github-copilot-sdk pre-installed"
- Live triage path tests remain in `tests/shared/Triage/Invoke-CopilotTriage.Timeout.Tests.ps1` (untouched, passing)

**Lesson:** Prefer structural assertions over test-expectation lowering. Orphaned modules are safe as long as we explicitly validate they stay orphaned.
