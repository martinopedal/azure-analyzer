## 2. Design principles applied

1. **Additive only.** No schema change, no wrapper change, no sanitiser change, no orchestrator change. Surface area unchanged; only normalizer enrichment.
2. **`New-FindingRow` remains the single emission path.** Patches add `-Dim` parameters; never construct `FindingRow` directly.
3. **Wrapper wins.** Every default is gated `if (-not $f.<Dim>)` so a richer wrapper output trumps the normalizer's heuristic.
4. **Vendor-fidelity first.** Defaults come from vendor metadata (e.g. PSRule baseline → `BaselineTags`, Scorecard CLI version → `ToolVersion`, Kubescape `hub.armosec.io` doc URL → `DeepLinkUrl`) — not invented values.
5. **Fixture-pinned.** Every enriched dim has a regression test with a sanitized fixture under `tests/fixtures/`.
